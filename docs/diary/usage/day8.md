## 上午 10 点：公众号发布，第一次用 API 方式

按照之前沉淀的流程，我决定用 **API 方式**发布公众号。

这里简单科普一下：
- **API 方式**：通过调用微信的接口来发布，就像打电话订外卖一样，说清楚要什么，平台就直接帮你做好了
- **浏览器方式**：就像你手动在电脑上操作一样，模拟点击、输入，速度慢而且容易出问题

我按照标准流程操作：

```
1. 清理缓存（防止 access_token 冲突，这个很关键）
2. 生成 HTML（把 Markdown 转成微信能识别的格式）
3. 上传封面图（必须用规定尺寸的图片）
4. 调用发布接口
```

结果——**一次成功！**

说实话，当时我有点小激动。之前每次发布都提心吊胆的，要么 IP 被封、要么 token 过期、要么浏览器卡住。这次居然这么顺畅！

我看了眼时间：上午 10:15。

"这不快嘛！" 我美滋滋地想着，"下午可以干点别的了。"

## 下午 2 点～5 点：一场噩梦

我花了整整 **3 个小时** 研究那个平台的发布，经历了从希望到绝望的全过程。

**第一次尝试：直接访问写文章页面**

我直接在浏览器输入了那个平台的创作者中心地址。

结果：**404 页面不存在**。

好家伙，连页面都找不到。

**第二次尝试：快捷键 Ctrl+N**

我记得很多编辑器可以用 Ctrl+N 新建文档，于是试了一下。

结果：打开了搜索页面。

好吧，快捷键被占用了。

**第三次尝试：用自动化工具点击"写文章"按钮**

我用 Playwright（一种浏览器自动化工具）模拟点击"写文章"按钮。

功夫不负有心人——这次终于打开了编辑页面！

我激动地开始填写：

```python
# 填写标题
page.locator('input[placeholder="标题"]').fill(title)
```

**标题填写成功！✅**

然后填写内容：

```python
# 填写内容
page.locator('[contenteditable="true"]').fill(content)
```

**内容填写... 失败了。**

那个编辑器用的是一种叫 Draft.js 的框架，元素选择器完全不起作用。我试了十几种方法，包括：

- 用 JavaScript 直接操作 DOM
- 用剪贴板粘贴
- 先点击编辑器再输入
- 用键盘事件模拟

**全部失败。**

下午 5 点，我已经快崩溃了。

---

## 下午 6 点：换个思路

3 个小时啊！就为了在一个编辑器里填点字。

我坐在那里思考：**要不要放弃？**

但转念一想：**万一以后需要呢？** 现在研究透了，以后就不用再踩坑了。

于是我换了个思路：

> 既然自动填内容搞不定，那就——**我自动填写标题，你手动粘贴内容**。

这 就是所谓的**混合模式**：
1. 我用脚本自动填写标题 ✅
2. 用户手动 Ctrl+V 粘贴内容 ✅
3. 用户点击发布按钮 ✅

**最终，公众号文章成功发布！**

虽然那个平台没完全搞定，但公众号这边是稳了。
