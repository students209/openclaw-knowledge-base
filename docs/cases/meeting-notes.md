# 案例：会议纪要助手

> ⭐⭐⭐ 难度：进阶 | ⏱ 预计时间：1小时

---

## 效果演示

自动整理会议内容：
- 识别会议要点
- 提取待办事项
- 生成会议纪要

---

## 实现思路

1. 接收会议记录（文字/语音转文字）
2. AI 分析内容
3. 结构化输出

---

## 核心配置

```json
{
  "name": "meeting-notes",
  "systemPrompt": "你是一个会议纪要助手。\n\n职责：\n1. 阅读会议记录\n2. 提取关键信息：议题、讨论、决策\n3. 列出待办事项：谁负责、截止时间\n4. 生成结构化会议纪要\n\n输出格式：\n- 会议基本信息\n- 讨论要点\n- 决策事项\n- 待办清单"
}
```

---

## 使用方式

### 输入会议记录

```
用户：今天下午3点开了产品评审会，张三介绍了新功能需求，李四提出技术实现方案，王五确认了上线时间。会议决定：1. 新功能4月1日上线 2. 张三负责前端，李四负责后端 待办：1. 张三：前端接口文档 4月1日前 2. 李四：后端开发 3月25日前
```

### AI 输出

```markdown
# 会议纪要

## 基本信息
- 日期：2026-03-13
- 主题：产品评审会

## 讨论要点
1. 新功能需求介绍
2. 技术实现方案
3. 上线时间确认

## 决策事项
- 新功能4月1日上线

## 待办清单
| 事项 | 负责人 | 截止 |
|------|--------|------|
| 前端接口文档 | 张三 | 4月1日 |
| 后端开发 | 李四 | 3月25日 |
```
