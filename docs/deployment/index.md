# 部署方案

生产环境部署指南。

---

## 部署选项

| 方式 | 难度 | 适合场景 |
|------|------|----------|
| [Docker](./docker) | ⭐ | 快速部署、容器化 |
| [服务器](./server) | ⭐⭐ | 传统服务器部署 |
| [群晖 NAS](./nas) | ⭐⭐ | NAS 用户 |
| [Render](./render) | ⭐ | 免费云平台 |
| [Railway](./railway) | ⭐ | 免费云平台 |
| [域名配置](./domain) | ⭐⭐ | 自定义域名+HTTPS |

---

## 选择建议

| 场景 | 推荐方案 |
|------|----------|
| 个人使用 | Docker / 本地 |
| 快速上线 | Render / Railway |
| 企业使用 | 服务器 + Docker |
| NAS 用户 | 群晖 Docker |

---

## 方案对比

### 免费平台

| 平台 | 免费额度 | 优点 | 缺点 |
|------|----------|------|------|
| [Render](./render) | 750小时/月 | 自动部署、HTTPS | 休眠后启动慢 |
| [Railway](./railway) | $5/月 | 按需付费 | 超额度收费 |
| [Vercel](https://vercel.com) | 100GB/月 | 速度快 | 仅前端 |

### 自建部署

| 方案 | 成本 | 优点 | 缺点 |
|------|------|------|------|
| [Docker](./docker) | 取决于服务器 | 可控性强 | 需要服务器 |
| [服务器](./server) | ¥30+/月 | 稳定 | 需要运维 |
| [群晖 NAS](./nas) | 取决于NAS | 数据本地 | 仅局域网 |

---

## 部署前准备

1. 确认 Node.js 22.12.0+
2. 准备域名（可选）
3. 选择服务器/平台
4. 了解基本命令

---

## 快速开始

```bash
# 方式一：Docker（推荐）
docker build -t openclaw .

# 方式二：服务器
npm install -g openclaw
openclaw gateway start

# 方式三：一键部署到 Render
# Fork 仓库 → 连接 Render → 自动部署
```
