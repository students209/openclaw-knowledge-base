# 训练流程

完整的训练流程。

---

## 步骤

### 1. 准备数据

将训练文档放入知识库目录：

```bash
# 创建知识库目录
mkdir -p ~/.openclaw/memory/my-knowledge

# 放入文档
cp your-document.md ~/.openclaw/memory/my-knowledge/
```

### 2. 检查状态

```bash
# 查看记忆状态
openclaw memory status
```

### 3. 索引文档

```bash
# 索引文档
openclaw memory index

# 或强制重新索引
openclaw memory index --force
```

### 4. 测试效果

```bash
# 搜索测试
openclaw memory search <你的专业问题>
```

---

## 常用命令

| 命令 | 说明 |
|------|------|
| `openclaw memory status` | 查看状态 |
| `openclaw memory status --deep` | 详细状态检查 |
| `openclaw memory index` | 索引文档 |
| `openclaw memory index --force` | 强制重新索引 |
| `openclaw memory search <query>` | 搜索 |

## 训练技巧

### 1. 文档质量
- 使用结构化的 Markdown
- 避免重复内容
- 使用清晰的专业术语

### 2. 分块策略
- 每个文档聚焦一个主题
- 避免过长的文档
- 适当拆分大文件

### 3. 持续更新
- 定期更新知识库
- 删除过时内容
- 添加新内容

## 实际案例

### 案例1：客服知识库

**场景**：训练一个回答产品问题的 AI

**数据准备**：
```
memory/product-faq/
├── 产品介绍.md      # 产品功能、定价
├── 常见问题.md      # FAQ 列表
├── 使用教程.md      # 使用步骤
├── 故障排除.md      # 常见问题解决
└── 联系方式.md      # 客服联系方式
```

**效果**：用户提问时，AI 会从这些文档中检索相关内容回答

## 高级技巧

### 1. 结构化文档

推荐的文件结构：
```markdown
# 主题标题

## 概述
一句话介绍

## 核心内容
详细说明...

### 子主题1
具体内容

### 子主题2
具体内容

## 常见问题
- Q: 问题1
- A: 答案1

## 参考资料
- 链接1
- 链接2
```

### 2. 关键词优化

在文档中适当重复关键术语，帮助检索：
- 首次出现时用全称
- 后续可用简称
- 适当使用同义词

### 3. 示例配合

理论+示例结合：
```markdown
## 如何使用

### 示例
实际使用示例代码或步骤

### 注意事项
使用中需要注意的点
```

## 常见问题

### 索引很慢

- 检查文档大小（建议单个文件 < 10MB）
- 减少文档数量
- 等待索引完成
- 使用 `openclaw memory index --force` 强制重新索引

### 搜索不到结果

- 确认文档已索引（`openclaw memory status`）
- 检查文档内容是否有意义
- 使用更通用的关键词
- 确认文档在正确目录

### 回答不准确

- 检查文档内容质量
- 增加相关文档
- 优化文档结构
- 尝试不同的提问方式
