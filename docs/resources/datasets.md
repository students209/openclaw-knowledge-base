# 数据集

训练用数据集的准备与示例。

---

## 一、数据集格式

### 1.1 支持的格式

| 格式 | 扩展名 | 说明 |
|------|---------|------|
| Markdown | .md | 推荐格式，支持富文本 |
| 纯文本 | .txt | 简单文本 |
| JSON | .json | 结构化数据 |
| CSV | .csv | 表格数据 |

### 1.2 格式示例

**Markdown 格式（推荐）**：
```markdown
# 文档标题

## 主题1
内容...

## 主题2
内容...
```

**JSON 格式**：
```json
{
  "question": "问题内容",
  "answer": "答案内容"
}
```

**CSV 格式**：
```csv
question,answer
问题,答案
```

## 二、数据准备原则

### 2.1 内容质量

| 要求 | 说明 |
|------|------|
| 准确性 | 内容必须正确可靠 |
| 完整性 | 逻辑完整，有始有终 |
| 专业性 | 使用规范术语 |
| 实用性 | 有实际应用价值 |

### 2.2 结构规范

| 要求 | 说明 |
|------|------|
| 主题明确 | 每个文件聚焦一个主题 |
| 层次清晰 | 标题层级合理 |
| 段落分明 | 内容分段清晰 |
| 重点突出 | 关键信息突出 |

### 2.3 内容避免

| 避免 | 原因 |
|------|------|
| 错别字 | 影响理解 |
| 过时信息 | 可能误导 |
| 重复内容 | 浪费资源 |
| 机密信息 | 安全风险 |

## 四、数据集存放位置

### 4.1 目录结构

```
~/.openclaw/memory/
├── my-knowledge/          # 主知识库
│   ├── faq/              # FAQ 数据
│   │   ├── product-faq.md
│   │   └── technical-faq.md
│   ├── docs/             # 文档数据
│   │   ├── product.md
│   │   └── company.md
│   └── notes/           # 笔记数据
│       └── learning.md
└── another-kb/           # 另一个知识库
    └── ...
```

### 4.2 多个知识库

```bash

├── product-kb/           # 产品知识库
```

## 五、数据集准备步骤

### 5.1 收集阶段

1. **来源确定**
   - 现有文档
   - 网站内容
   - 聊天记录
   - 原创内容

2. **内容收集**
   - 复制内容
   - 下载文件
   - 导出数据

### 5.2 整理阶段

1. **格式转换**
   - 转为 Markdown
   - 清理格式

2. **结构规划**
   - 分类整理
   - 命名规范

### 5.3 导入阶段

1. **放入目录**
```bash
cp -r my-data/* ~/.openclaw/memory/my-knowledge/
```

2. **执行索引**
```bash
openclaw memory index
```

3. **验证效果**
```bash
openclaw memory search "关键词"
```

## 七、高级技巧

### 7.1 优化搜索

在文档中适当重复关键词：
```markdown
# AI 助手
AI助手是...（首次出现用全称）

我们这个AI助手...（后续可用简称）

也叫 AI-Agent...（可加入同义词）
```

### 7.2 链接引用

使用交叉引用：
```markdown
# 相关文档
- [产品介绍](./product.md)
- [价格说明](./pricing.md)
- [联系方式](./contact.md)
```
