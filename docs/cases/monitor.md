# 案例：信息监控助手

> ⭐⭐⭐⭐ 难度：高级 | ⏱ 预计时间：2小时

---

## 效果演示

自动化监控：
- 网站内容变化
- RSS 订阅更新
- 价格变动
- 自动推送通知

---

## 实现思路

1. 安装相关技能
2. 配置监控目标
3. 设置定时检查
4. 自动推送通知

---

## 核心配置

### 安装技能

```bash
# 安装监控相关技能
openclaw skills install tavily
openclaw skills install rss-reader  # 如果有
```

### 任务配置

```json
{
  "name": "info-monitor",
  "systemPrompt": "你是一个信息监控助手。\n\n职责：\n1. 监控指定网站/RSS 的更新\n2. 发现新内容时提取关键信息\n3. 将重要信息推送给用户\n\n关注领域：\n- 技术资讯\n- 产品更新\n- 竞品动态",
  "skills": ["tavily"]
}
```

## 监控场景

### 场景1：竞品动态监控

```
用户：帮我监控以下竞品官网，有更新时通知我
- https://example.com/blog
- https://competitor.com/news

助手：已设置监控，每天检查一次，有更新会通知你。
```

### 场景2：价格监控

```
用户：监控 iPhone 15 Pro 价格，低于 7000 元时通知我

助手：已设置价格监控，价格变化会及时通知你。
```

### 场景3：资讯订阅

```
用户：每天早上8点推送 AI 行业最新资讯

助手：已设置，每天8点推送当天热门资讯。
```

---

## 进阶：自动化流程

结合 cron 实现完全自动化：

```bash
# 每天早上8点推送资讯
cron add --schedule "0 8 * * *" --task info-monitor --message "推送今天的行业资讯"

# 每小时检查价格
cron add --schedule "0 * * * *" --task price-monitor --message "检查价格变动"
```

## 常见问题

### 1. 监控频率

- 太频繁：可能被封 IP
- 太慢：错过重要信息
- 建议：每小时或每天一次

### 2. 推送方式

支持推送到的平台：
- Discord
- Telegram
- 飞书
- 邮件
