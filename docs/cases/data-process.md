## 实现思路

1. **安装技能**：安装文件处理相关技能
2. **准备数据**：准备要处理的 Excel/CSV 文件
3. **编写提示词**：定义数据处理规则
4. **执行任务**：让 AI 处理数据

## 核心配置

### 安装技能

```bash
# 安装文件处理技能
openclaw skills install nano-pdf

# 安装编码技能
openclaw skills install coding-agent
```

### 任务配置

```json
{
  "name": "data-processor",
  "systemPrompt": "你是一个数据处理助手。\n\n职责：\n1. 读取用户提供的数据文件\n2. 按要求处理数据（筛选、计算、统计等）\n3. 输出处理结果或生成报告\n\n支持格式：\n- CSV\n- Excel (.xlsx)\n- JSON\n\n输出格式：\n- 处理后的数据文件\n- 统计报告（Markdown）",
  "skills": ["nano-pdf", "coding-agent"]
}
```

## 完整示例

### 示例：销售数据分析

用户上传销售数据，AI 自动分析：

**输入数据**（sales.csv）：
```csv
日期,产品,数量,金额
2024-01-01,产品A,100,5000
2024-01-02,产品B,50,3000
...
```

**处理请求**：
```
帮我分析这份销售数据，计算：
1. 每个产品的总销量
2. 每个产品的总销售额
3. 销售额最高的产品
```

**AI 输出**：
```markdown
## 销售数据分析报告

### 总销量
- 产品A：500件
- 产品B：300件
- 产品C：200件

### 总销售额
- 产品A：¥25,000
- 产品B：¥18,000
- 产品C：¥12,000

### 销售额最高
产品A，销售额 ¥25,000
```

## 使用方式

```bash
# 创建任务
openclaw task create data-processor

# 配置后启动
openclaw gateway start

# 使用方式
用户：帮我分析 attachments/sales.xlsx 这份数据
AI：（分析并输出报告）
```

## 进阶：定时处理

可以设置定时任务自动处理：

```bash
# 每天早上9点处理数据
cron job add --schedule "0 9 * * *" --task data-processor --message "处理昨天的销售数据"
```

## 常见问题

### 1. 无法读取文件

- 确认文件格式正确
- 检查文件权限
- 确认文件路径正确

### 2. 处理太慢

- 减少数据量
- 使用更快的模型
- 分批处理
