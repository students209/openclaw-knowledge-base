## 什么是任务？

任务（Task）是 OpenClaw 的核心概念。每个任务代表一个 AI Agent，可以：
- 拥有独立的配置
- 执行特定的功能
- 使用特定的技能

你可以创建多个任务，每个任务负责不同的场景。

---

## 创建任务

### 通过 CLI 创建

```bash
# 创建新任务
openclaw task create my-first-task
```

### 通过 Web 界面创建

1. 打开 http://localhost:3000
2. 点击「新建任务」
3. 输入任务名称和描述
4. 选择模型和技能

---

## 与 AI 对话

### 通过 Web 界面

1. 打开 http://localhost:3000
2. 选择任务
3. 在输入框中输入问题
4. 发送消息，等待回复

### 通过 CLI

```bash
# 与默认任务对话
openclaw chat "你好，请介绍一下自己"

# 与指定任务对话
openclaw chat -t my-first-task "你好"
```

---

## 任务配置

每个任务可以独立配置：

```json
{
  "name": "my-first-task",
  "description": "我的第一个 AI 助手",
  "model": {
    "provider": "bailian",
    "model": "MiniMax-M2.5"
  },
  "systemPrompt": "你是一个友好的 AI 助手，帮助用户解决问题。",
  "skills": ["weather", "summarize"]
}
```

### 配置说明

| 配置项 | 说明 |
|--------|------|
| name | 任务名称（唯一） |
| description | 任务描述 |
| model | 使用的模型 |
| systemPrompt | 系统提示词，定义 AI 的角色和行为 |
| skills | 启用的技能列表 |

---

## 系统提示词

系统提示词（System Prompt）用于定义 AI 的行为和角色。

### 示例：客服助手

```
你是一个专业的客服助手，负责回答用户关于产品的问题。
- 语气友好、专业
- 回答简洁明了
- 如不确定，请如实说明
```

### 示例：写作助手

```
你是一个专业的文案写作者，擅长各种类型的文章创作。
- 语言生动有趣
- 结构清晰
- 根据用户需求调整风格
```

## 实战：创建一个问答助手

让我们创建一个简单的问答助手：

### 步骤 1：创建任务

```bash
openclaw task create qa-assistant
```

### 步骤 2：配置系统提示词

```bash
openclaw task edit qa-assistant
```

设置系统提示词：

```
你是一个知识渊博的问答助手，专门回答用户的技术问题。
- 回答要准确、详细
- 如果不确定答案，请说明不确定
- 可以适当举例说明
```

### 步骤 3：测试对话

```bash
openclaw chat -t qa-assistant "什么是 AI Agent？"
```

## 常见问题

### 1. AI 不回答问题

- 检查 API Key 是否配置正确
- 检查模型是否可用
- 查看日志：`openclaw gateway logs`

### 2. 回答不符合预期

- 调整系统提示词
- 检查任务配置
- 尝试不同的模型

### 3. 响应太慢

- 检查网络连接
- 尝试更快的模型
- 检查 API 配额
