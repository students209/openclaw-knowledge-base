## 为什么要训练 AI？

通用 AI 模型虽然强大，但：
- ❌ 不了解你的专业知识
- ❌ 不熟悉你的业务场景
- ❌ 不知道你的文档内容

通过训练，你可以让 AI：
- ✅ 成为领域专家
- ✅ 回答专业问题
- ✅ 引用你的文档

## 训练方式

OpenClaw 提供两种训练方式：

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| 文档训练 | 上传文档让 AI 学习 | 知识库问答 |
| 记忆训练 | 管理长期记忆 | 个性化对话 |

---

## 文档训练

### 步骤 1：准备文档

将需要 AI 学习的文档放入训练目录：

```bash
# 默认训练目录
~/.openclaw/training/
```

支持的格式：
- `.txt` - 纯文本
- `.md` - Markdown
- `.pdf` - PDF（需要技能）
- `.docx` - Word（需要技能）

### 步骤 2：执行训练

```bash
# 训练所有文档
openclaw train

# 重新训练
openclaw train --rebuild

# 查看训练状态
openclaw memory status
```

### 步骤 3：测试

训练完成后，AI 会自动使用你的文档作为参考：

```
用户：关于我们的产品政策是什么？
AI：（从你的文档中检索并回答）根据我们的产品政策文档...
```

## 记忆训练

记忆训练让你可以管理 AI 的长期记忆。

### 基本命令

```bash
# 查看记忆状态
openclaw memory status

# 重新索引
openclaw memory index

# 搜索记忆
openclaw memory search <关键词>

# 添加记忆
openclaw memory add "用户喜欢在下午2点开会"

# 列出所有记忆
openclaw memory list

# 删除记忆
openclaw memory remove <id>
```

### 记忆文件位置

- `memory/YYYY-MM-DD.md` - 每日记忆
- `MEMORY.md` - 长期记忆

## 数据准备

### 文档质量指南

好的训练文档应该：
- ✅ 内容准确、权威
- ✅ 结构清晰
- ✅ 避免敏感信息
- ✅ 定期更新

### 文档结构建议

```
# 产品介绍

## 产品概述
我们的产品是...

## 功能特点
1. 功能一：...
2. 功能二：...
3. 功能三：...

## 使用方法
1. 第一步：...
2. 第二步：...
...
```

### 推荐的数据量

| 场景 | 建议数据量 |
|------|------------|
| 入门级 | 10-50 页 |
| 进阶级 | 50-200 页 |
| 专业级 | 200+ 页 |

## 训练优化

### 调整系统提示词

在任务配置中添加强调：

```json
{
  "systemPrompt": "你是公司的产品客服，熟悉公司所有产品文档。请根据文档内容回答用户问题。"
}
```

### 使用技能增强

安装知识管理相关技能：

```bash
# 安装相关技能
openclaw skills install notion
openclaw skills install qmd
```

## 常见问题

### 1. 训练太慢

- 减少文档数量
- 使用更短的文档
- 避免 PDF（需要额外处理）

### 2. AI 不使用文档内容

- 检查文档是否在训练目录
- 确认训练已执行
- 调整系统提示词强调使用文档

### 3. 文档内容不安全

- 不要在文档中包含敏感信息
- 使用敏感信息脱敏工具
- 定期清理不需要的文档
