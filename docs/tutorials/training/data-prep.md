# 数据准备

准备训练数据，让 AI 学习你的专业知识。

---

## 一、支持的数据格式

### 1.1 格式对比

| 格式 | 说明 | 推荐程度 | 处理难度 |
|------|------|----------|----------|
| **Markdown (.md)** | 最佳选择，支持格式 | ⭐⭐⭐⭐⭐ | 简单 |
| **TXT (.txt)** | 纯文本支持 | ⭐⭐⭐⭐ | 简单 |
| **PDF (.pdf)** | 支持，需 OCR 处理 | ⭐⭐⭐ | 中等 |
| **JSON (.json)** | 结构化数据 | ⭐⭐⭐ | 简单 |
| **HTML (.html)** | 网页内容 | ⭐⭐ | 中等 |
| **Word (.docx)** | 部分支持 | ⭐⭐ | 较难 |

**推荐格式：Markdown**

---

## 二、数据要求

### 2.1 文件大小

| 指标 | 建议值 | 最大值 |
|------|--------|--------|
| 单个文件 | < 1MB | 10MB |
| 总知识库 | < 50MB | 100MB |
| 单个文档字数 | 1000-5000字 | 无严格限制 |

### 2.2 内容要求

**必须满足**：
- 内容清晰、结构化
- 避免大量重复内容
- 使用规范的术语
- 有实际价值的内容

**建议避免**：
- 无意义的文字
- 过时的信息
- 未经验证的内容
- 机密/敏感信息

## 三、数据目录结构

### 3.1 推荐结构

```
~/.openclaw/memory/
├── product-faq/              # 主题文件夹
│   ├── 01-intro.md          # 产品介绍
│   ├── 02-pricing.md       # 定价
│   ├── 03-faq.md           # 常见问题
│   └── index.md            # 目录（可选）
├── company-kb/              # 另一个知识库
│   ├── 01-about.md
│   ├── 02-culture.md
│   └── 03-process.md
└── personal/               # 个人知识
    ├── 01-notes/
    └── 02-ideas/
```

### 3.2 命名规范

```bash
# 使用有意义的名称
good: 01-product-intro.md
good: 02-pricing-plan.md

# 避免
bad: 123.md
bad: 新文档(1).md
```

## 四、Markdown 格式详解

### 4.1 推荐的文档结构

```markdown
# 主题标题（一级标题）

> 一句话概括这个主题

## 概述
简短介绍

## 核心内容

### 要点1
详细说明...

### 要点2
详细说明...

## 常见问题

**Q: 问题1**
A: 答案1

**Q: 问题2**
A: 答案2

## 参考资料
- [链接1](url1)
- [链接2](url2)
```

### 4.2 格式示例

```markdown
# 智能客服系统

> 帮助企业快速搭建智能客服系统

## 概述
智能客服系统是一种基于 AI 技术的自动化客服解决方案...

## 核心功能

### 1. 智能问答
系统可以自动回答用户问题，覆盖 90% 常见问题

### 2. 人机协作
复杂问题自动转人工，确保服务质量

### 3. 数据分析
实时监控客服数据，生成可视化报表

## 常见问题

**Q: 支持哪些渠道？**
A: 支持网页、微信、APP 等主流渠道

**Q: 如何收费？**
A: 按坐席数量收费，详见官网定价

## 价格
- 免费版：5 个坐席
- 专业版：99 元/坐席/月
- 企业版：联系销售

## 竞品对比
| 功能 | 我们的产品 | 竞品A | 竞品B |
|------|------------|-------|-------|
| 智能问答 | ✅ | ✅ | ⚠️ |
| 人机协作 | ✅ | ✅ | ✅ |
| 数据分析 | ✅ | ⚠️ | ✅ |

## 参考资料
- 官网：https://example.com
- 文档：https://docs.example.com
```

## 五、数据准备步骤

### 5.1 步骤流程

```
1. 确定主题 → 2. 收集内容 → 3. 整理结构 → 4. 格式转换 → 5. 质量检查 → 6. 放入目录
```

### 5.2 详细操作

**步骤1：确定主题**
- 明确知识库的用途
- 确定核心内容范围

**步骤2：收集内容**
- 从现有文档中提取
- 从网上搜集资料
- 原创内容

**步骤3：整理结构**
- 按主题分类
- 规划目录结构

**步骤4：格式转换**
- 转换为 Markdown
- 规范格式

**步骤5：质量检查**
- 检查错别字
- 验证内容准确性
- 确保结构清晰

**步骤6：放入目录**
```bash
# 创建目录
mkdir -p ~/.openclaw/memory/你的知识库

# 复制文件
cp *.md ~/.openclaw/memory/你的知识库/
```

## 七、数据转换工具

### 7.1 转换为 Markdown

| 工具 | 用途 |
|------|------|
| Pandoc | 各种格式转 Markdown |
| VS Code | 预览和编辑 |
| Notion | 导出为 Markdown |
| Google Docs | 导出为 Markdown |

### 7.2 PDF 转 Markdown

```bash
# 使用 pdftotext（需要安装 poppler）
pdftotext input.pdf output.txt

# 使用在线工具
# https://pdf2md.com/
```

### 7.3 Word 转 Markdown

```bash
# 使用 Pandoc
pandoc input.docx -o output.md

# 使用在线工具
# https://word2md.com/
```

## 八、常见问题

### 8.1 数据格式问题

**Q: PDF 文件无法读取？**
A: 建议转换为 Markdown 格式，或使用 OCR 工具提取文本

**Q: 文件太大怎么办？**
A: 拆分为多个小文件，每个文件聚焦一个主题

### 8.2 内容问题

**Q: 内容从哪里获取？**
A: 现有文档、官网信息、GitHub README、行业文章等

**Q: 内容有错误怎么办？**
A: 及时更新，知识库需要定期维护
