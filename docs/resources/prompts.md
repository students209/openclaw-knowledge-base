# 提示词模板

常用提示词模板，提升工作效率。

---

## 一、写作助手

### 1.1 文章撰写（详细版）

```
请帮我写一篇关于[主题]的文章，要求：

## 基本要求
- 字数：约[数字]字
- 风格：[正式/轻松/专业/幽默]
- 目标读者：[描述目标读者]

## 结构要求
- 开头：引入主题，吸引读者
- 主体：[数字]个主要观点，每个观点有案例支持
- 结尾：总结+行动号召

## 特殊要求
- [如有特殊要求请说明]
- 避免：[需要避免的内容]
```

### 1.2 社交媒体文案

**小红书种草文案**
```
帮我写一篇小红书种草文案，推广[产品/服务]：

产品亮点：1、2、3
使用场景：[描述]
目标人群：[描述]
风格：真实、有感染力
字数：300-500字
必须包含：开头钩子、产品优势、个人体验、总结推荐
```

**微信公众号推文**
```
写一篇公众号推文，主题是[主题]：

文章类型：[种草/干货/故事/热点评论]
风格：[专业/亲切/犀利]
结构：热点引入→核心观点→案例分析→总结升华
字数：1500-2000字
```

**知乎回答**
```
回答问题：[问题全文]

要求：
- 专业、客观、有深度
- 包含实际案例或数据
- 适当引用权威来源
- 观点鲜明，论证充分
```

### 1.3 商务文档

**商业计划书**
```
帮我写[项目名称]的商业计划书：

## 项目概述
- 痛点：[解决什么问题]
- 解决方案：[提供什么价值]
- 目标用户：[谁会使用]

## 市场分析
- 市场规模：[数据]
- 竞争分析：[主要竞品]

## 商业模式
- 收入来源：[如何赚钱]
- 定价策略：[价格]

## 团队介绍
- 核心成员：[简介]
```

## 二、代码助手

### 2.1 代码审查

```
你是一位资深代码审查员，请审查以下[语言]代码：

## 代码
```[语言]
[粘贴代码]
```

## 审查要求
1. 找出潜在的 bug 和逻辑错误
2. 指出安全问题（如 SQL 注入、XSS 等）
3. 建议代码优化方案（性能、可读性）
4. 评估代码是否符合最佳实践

## 输出格式
- 问题列表（严重/警告/建议）
- 改进后的代码示例
```

### 2.2 代码生成

```
用 [语言] 实现 [功能]：

## 需求描述
[详细描述需要实现的功能]

## 输入
[输入数据格式]

## 输出
[输出数据格式]

## 要求
- 遵循 [语言] 最佳实践
- 添加中文注释
- 考虑错误处理
- 性能最优

## 上下文
[如有特殊上下文请说明]
```

### 2.3 技术文档

```
为以下代码生成技术文档：

## 代码
```[语言]
[代码]
```

## 需要生成
1. 功能说明（一句话）
2. 使用方法（参数说明）
3. 示例代码
4. 注意事项
5. 常见问题
```

分析以下数据，给出洞察：

## 需要分析

## 期望结果
[描述想要的结果]

## 四、学习助手

### 4.1 概念解释

```
用通俗易懂的语言解释 [概念]：

## 要求
- 目标受众：[完全不懂的小白 / 有基础的人 / 专业人士]
- 使用比喻/类比帮助理解
- 核心要点不超过 3 点
- 提供 1-2 个实际应用场景

## 输出结构
1. 一句话概括
2. 详细解释
3. 比喻/例子
4. 实际应用
5. 延伸学习资源
```

### 4.2 学习总结

```
总结 [主题] 的核心内容：

## 目标
- 用一句话概括这个主题
- 列出 5 个关键知识点
- 每个知识点用通俗语言解释

## 应用场景
- 这个知识可以应用在哪些场景？

## 学习建议
- 初学者从哪开始？
- 进阶需要学什么？
```

### 4.3 面试准备

```
帮我准备 [岗位] 面试：

## 公司/岗位
[公司名] - [岗位名]

## 需求
- 核心技术问题及答案
- 行为面试题及回答思路
- 薪资谈判建议

## 我的背景
[简要介绍你的背景]
```

## 五、产品经理

### 5.1 需求分析

```
分析以下需求：

## 需求描述
[粘贴需求]

## 分析要求
1. 目标用户是谁？
2. 核心痛点是什么？
3. 优先级评估（must-have / should-have / nice-to-have）
4. 技术实现复杂度
5. 潜在风险

## 输出
- 需求评审意见
- 改进建议
- 替代方案
```

### 5.2 用户调研

```
设计用户调研方案：

## 调研目标
[要了解什么]

## 调研对象
[目标用户特征]

## 调研方法
- [问卷/访谈/观察/数据分析]

## 问题清单
[列出要问的问题]

## 输出
- 调研报告模板
- 数据分析建议
```

## 六、自媒体运营

### 6.1 内容策划

```
策划一周内容：

## 账号定位
[账号名称] - [领域] - [目标人群]

## 本周主题
[主题方向]

## 需要
1. 7 天内容选题
2. 每篇的角度和切入點
3. 标题建议
4. 发布时间建议
```

### 6.2 爆款分析

```
分析这个[标题/内容]为什么能火：

## 内容
[粘贴内容或链接]

## 分析维度
1. 开头钩子（如何吸引点击）
2. 内容结构（如何留住读者）
3. 情绪价值（提供了什么情绪）
4. 传播因子（为什么让人想分享）
5. 可复用的点

## 结论
- 3 个可复用的技巧
- 应用到自己内容的建议
```

## 七、效率工具

### 7.1 日程管理

```
帮我规划 [日期] 的工作：

## 当前任务
[任务列表]

## 限制
- 工作时间：[时间段]
- 优先级：[重要/紧急]
- 预估时长

## 输出
- 时间块安排
- 执行顺序建议
- 注意事项
```

### 7.2 问题解决

```
分析并解决这个问题：

## 问题
[描述问题]

## 背景
[相关背景信息]

## 已尝试的方法
[如有过]

## 需要
1. 根本原因分析
2. 解决方案（至少 2 个）
3. 推荐方案及理由
4. 预防措施
```
